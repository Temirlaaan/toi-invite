# Task: Template System & Gallery

## Description
Define the template JSON configuration format, seed the database with 3 wedding templates, and build the public template gallery page with filtering by event type.

## Acceptance Criteria
- [ ] Template JSON config schema defined and documented (sections, fields, layout, color scheme)
- [ ] 3 wedding templates seeded via Prisma seed script with realistic config
- [ ] Template gallery page (`/[locale]/templates`) with grid layout and thumbnail cards
- [ ] Filter by event type (wedding, anniversary, kids — only wedding populated for now)
- [ ] Template preview page (`/[locale]/templates/[slug]`) showing full template render
- [ ] Templates display: thumbnail, name, category badge (free/standard/premium), event type
- [ ] Gallery is responsive (mobile-first grid)
- [ ] Page is server-rendered for SEO

## Technical Details
- Template `configJson` structure:
  ```json
  {
    "sections": ["hero", "details", "countdown", "gallery", "map", "rsvp", "gift"],
    "theme": { "primaryColor": "#...", "fontFamily": "..." },
    "fields": {
      "hero": { "backgroundImage": true, "title": true, "subtitle": true },
      "details": { "date": true, "time": true, "venue": true, "dressCode": true },
      "gallery": { "maxPhotos": 12 },
      "gift": { "kaspiQr": true }
    },
    "layout": "scroll-vertical"
  }
  ```
- Seed script: `prisma/seed.ts` with 3 distinct wedding styles (classic, modern, minimal)
- Gallery uses server component with search params for filtering
- Template preview renders the template with placeholder/demo data
- Thumbnail images stored as static assets initially (no MinIO needed for template thumbnails)
