# Task: Project Setup & Infrastructure

## Description
Initialize the Next.js + TypeScript project with all core dependencies and infrastructure configuration. This is the foundation task — everything else builds on it.

## Acceptance Criteria
- [ ] Next.js 14+ App Router project initialized with TypeScript
- [ ] Tailwind CSS configured with shadcn/ui components installed
- [ ] Prisma ORM installed and configured (connection string via env var)
- [ ] MinIO S3 SDK (`@aws-sdk/client-s3`) installed and configured with env vars
- [ ] `next-intl` configured with `kk` and `ru` locales, locale routing via `[locale]` segment
- [ ] `docker-compose.yml` with services: `app`, `postgres`, `minio`, `nginx`
- [ ] Nginx config: reverse proxy to Next.js, SSL placeholder, static cache headers
- [ ] Dockerfile using Next.js standalone output
- [ ] `.env.example` with all required environment variables documented
- [ ] Project runs locally with `docker-compose up` and `npm run dev`

## Technical Details
- Use `create-next-app` with App Router + TypeScript + Tailwind
- shadcn/ui init with default theme
- next-intl middleware for locale detection and routing
- Directory structure:
  ```
  src/
    app/[locale]/          # All pages under locale
    components/            # Shared UI components
    lib/                   # Utilities (db, s3, auth config)
    i18n/                  # Translation files (kk.json, ru.json)
  prisma/
    schema.prisma
  docker/
    nginx.conf
    Dockerfile
  docker-compose.yml
  ```
- PostgreSQL 16, MinIO latest in docker-compose
- Nginx proxies port 3000, exposes port 80/443
